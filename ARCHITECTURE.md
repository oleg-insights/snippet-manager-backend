# Архитектура Snippet Manager

## 1. Модель данных и бизнес-логика

### Теги и шаблоны

- **Шаблон** - структурированный набор текстовых и других данных (код, список, изображение и т.д.), например, письмо
    - `authorId` - автор шаблона
    - `isPublic` - видимость шаблона для других пользователей (true/false)
- **Тег** - основная единица категоризации. У каждого тега есть автор (`authorId`) и владелец (`scopeUserId`)
    - `scopeUserId` - определяет доступность тега (null - все, id пользователя - один пользователь)

- **Жизненный цикл тега:**
    - При создании шаблона автор может использовать публичные теги или создать свои приватные. Если указывается имя уже существующего публичного тега, будет использован именно он
    - При публикации шаблона **все приватные теги этого шаблона становятся публичными** (`scopeUserId = null`). Тег становится доступным всем
    - При снятии с публикации или удалении шаблона для каждого тега проверяется привязка к другим шаблонам. Если тег больше не используется ни в одном публичном шаблоне и принадлежит только одному пользователю, он снова становится приватным для него. Если не используется никем — удаляется

### Иерархия тегов (динамическая)

- Вместо статического поля `parentId`, которое сложно синхронизировать, используется **динамическая рекомендация родительских тегов** для дочернего набора
- При поиске шаблонов вычисляются теги, которые присутствуют в **каждом** шаблоне из **всей** текущей выборки и при этом имеют дополнительные шаблоны вне выборки. Такие теги предлагаются как "обобщающие" (например, при выборе "mysql" рекомендованным родительским станет "sql")
- Фронтенд, кликая по рекомендованному тегу, заменяет им дочерний набор, расширяя поиск

## 2. Аутентификация и безопасность

- **JWT**: Access-токен (15 мин) и Refresh-токен (30 дней)
- **Логаут**: При выходе access-токен заносится в **blacklist Redis** на время его жизни. Связанная сессия удаляется
- **Защита роутов**: `JwtAuthGuard` проверяет валидность токена и его отсутствие в blacklist. `RolesGuard` ограничивает доступ к админ-роутам
- **Безопасность данных**: Пароли хешируются `bcrypt`. Контент шаблонов проходит рекурсивную очистку через `xss` для предотвращения XSS. Глобально включены `Helmet` и `CORS`. `Rate limiting` подключается выборочно.

## 3. Поиск и фильтрация (`GET /templates`)

- Поиск осуществляется по набору выбранных тегов: шаблон должен быть привязан ко **всем выбранным**
- Доступность тегов для поиска определяется динамически: тег виден, если существует хотя бы один доступный пользователю шаблон с этим тегом
- Ответ API содержит не только `data`, но и:
    - `availableTags` — теги, которые можно добавить к фильтру для сужения поиска
    - `selectedTags` — текущие выбранные теги
    - `suggestedParents` — рекомендованные родительские теги для расширения поиска

## 4. Структура проекта

- `src/auth` — аутентификация и авторизация (стратегии, гарды, декораторы)
- `src/common` — переиспользуемые модули (валидация, DTO, утилиты)
- `src/common/config` — конфигурация приложения и переменных окружения
- `src/prisma` — сервис Prisma, seed
- `src/redis` — сервис для работы с Redis
- Остальные модули (`users`, `templates`, `tags`, `admin`) — реализация логики для этих сущностей
